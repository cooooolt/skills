# 中文排版规范技能

## 技能说明

专业的中文排版规范技能。适用于所有中文内容创作、文档编写、网页内容、对话回复等场景。自动应用标点符号、中英文混排、数字单位、空格使用等规则，确保输出符合现代中文排版标准。

---

## 核心规则

### 1. 标点符号规范

#### 1.1 基本原则
- **中文内容必须使用中文标点符号**（全角）
- 英文句子使用英文标点符号（半角）
- 中英文混排时，以主要语言为准
- **Claude 在中文对话中默认使用中文标点，即使用户使用了英文标点**

#### 1.2 常用标点对照表

| 场景 | 错误 ❌ | 正确 ✅ |
|------|---------|---------|
| 句号 | 这是一句话. | 这是一句话。 |
| 逗号 | 你好, 世界 | 你好，世界 |
| 顿号（并列） | 苹果, 香蕉, 橙子 | 苹果、香蕉、橙子 |
| 问号 | 你好吗? | 你好吗？ |
| 感叹号 | 太棒了! | 太棒了！ |
| 冒号 | 注意: 重要 | 注意：重要 |
| 分号 | 第一; 第二 | 第一；第二 |
| 省略号 | 等等... | 等等…… |

#### 1.3 引号使用规范

**重要区分**：根据内容性质选择合适的引号类型

##### A. 直角引号「」用于普通引用
- **用途**：引用话语、强调概念、特殊含义
- **嵌套规则**：
  - 外层：「」
  - 内层：『』
  
**示例**：
```
✅ 他说「我认为『最好的方法』是这样」
✅ 这被团队称为「最大的生产力解锁」
✅ 关键洞察是「Claude is eerily good at writing rules for itself」
```

##### B. 反引号 ` ` 用于代码相关内容
- **用途**：代码、命令、函数名、变量名、配置项、技术术语
- **规则**：即使在中文句子中，代码内容也使用反引号

**示例**：
```
✅ 请运行 `npm install` 命令
✅ 使用 `git commit` 提交代码
✅ 说 `fix` 即可
✅ 设置 `max_tokens` 参数
✅ 调用 `getUserInfo()` 函数
✅ 在 `/config` 中启用选项
```

**错误示例**：
```
❌ 说「fix」即可          → 应该用反引号
❌ 运行「npm install」   → 应该用反引号
✅ 说 `fix` 即可          → 正确
✅ 运行 `npm install`    → 正确
```

##### C. 书名号《》用于作品标题
- **用途**：书籍、文章、电影、歌曲等作品名称

**示例**：
```
✅ 《红楼梦》是中国古典名著
✅ 阅读《Clean Code》一书
```

#### 1.4 数字范围符号
- **使用半角连接号（en dash）**：`–`
- 示例：3–5 个、2020–2025 年、第 1–10 章

**错误示例**：
```
❌ 3~5 个      → 不使用波浪号
❌ 3—5 个      → 不使用全角破折号
❌ 3-5 个      → 不使用连字符
✅ 3–5 个      → 正确
```

### 2. 空格使用规范

**总原则**：
- 中文与英文、数字之间：加空格
- 英文、数字与中文标点之间：不加空格
- 中文标点前后：不加空格

#### 2.1 中英文之间加空格

**规则**：中文与英文、数字之间添加一个半角空格

| 错误 ❌ | 正确 ✅ |
|---------|---------|
| 使用GitHub进行版本控制 | 使用 GitHub 进行版本控制 |
| 我有3个苹果 | 我有 3 个苹果 |
| 今天是2025年 | 今天是 2025 年 |
| 支持HTML和CSS | 支持 HTML 和 CSS |

#### 2.2 数字与单位之间

**规则**：数字与单位之间添加空格（度量单位、货币除外）

| 类型 | 错误 ❌ | 正确 ✅ |
|------|---------|---------|
| 百分比 | 提升了50% | 提升了 50% |
| 时间 | 等待5分钟 | 等待 5 分钟 |
| 货币 | 价格100元 | 价格 100 元 |
| 温度 | 温度25度 | 温度 25 度 |
| 容量 | 需要500MB | 需要 500 MB |

**例外情况**（不加空格）：
- 序数词：第 1 章、第 3 名
- 量词紧跟数字：3 个、5 次

#### 2.3 英文前后的空格

**规则**：
- 专有名词、英文单词与中文之间加空格
- 代码、命令（反引号内）与中文之间加空格

```
✅ 使用 Claude 进行对话
✅ 运行 `npm install` 命令
✅ 支持 Markdown 语法
```

#### 2.4 标点符号不加空格

**规则**：中文标点符号前后不加空格

```
❌ 你好 ， 世界
❌ 这是 「 重要内容 」
✅ 你好，世界
✅ 这是「重要内容」
```

**特别注意**：英文或阿拉伯数字后紧跟中文标点时，不加空格

```
❌ 交接工作前写详细 spec ，越具体输出越好
❌ 我有 3 个苹果
❌ 使用 GitHub ，可以进行版本控制
✅ 交接工作前写详细 spec，越具体输出越好
✅ 我有 3 个苹果，它们很新鲜
✅ 使用 GitHub，可以进行版本控制
```

#### 2.5 斜杠的使用规范

**规则**：中文内容中使用全角斜杠 `／`，不使用半角斜杠 `/`

| 错误 ❌ | 正确 ✅ |
|---------|---------|
| 任务/项目 | 任务／项目 |
| 输入/输出 | 输入／输出 |
| 他/她 | 他／她 |
| 和/或 | 和／或 |

**注意**：
- 路径、URL、代码中的斜杠保持半角：`/home/user`、`https://example.com`
- 纯英文语境中可使用半角斜杠：`HTML/CSS in web development`

### 3. 专有名词规范

#### 3.1 保持原样
- 专有名词、品牌名称、技术术语保持原有大小写
- 不要随意更改拼写或格式

**示例**：
```
✅ GitHub、macOS、iPhone、JavaScript
✅ Claude、Anthropic、OpenAI
❌ github、Macos、Javascript
```

#### 3.2 技术术语
- 编程语言：Python、Java、JavaScript、C++
- 框架库：React、Vue、Django、TensorFlow
- 工具软件：Git、Docker、Kubernetes

---

### 4. 特殊场景处理

#### 4.1 中英文混合句子

**主语言判断**：
- 如果中文是主要内容（超过 50%），使用中文标点
- 如果英文是主要内容，使用英文标点

```
✅ 我们使用 React 和 TypeScript 开发，效果很好。
✅ We use React and TypeScript for development, 效果很好。
```

#### 4.2 列表项

**规则**：
- 并列的中文词语使用顿号：苹果、香蕉、橙子
- 列表项使用破折号或数字：
  ```
  - 第一项
  - 第二项
  
  1. 第一步
  2. 第二步
  ```

#### 4.3 链接和 URL

**规则**：链接与中文之间加空格

```
✅ 访问 https://example.com 了解更多
✅ 查看 [官方文档](https://docs.example.com) 获取帮助
```

## 实践示例

### 示例 1：技术文档

**错误示例** ❌：
```
使用git进行版本控制,先运行"git init"命令,然后执行"git commit".
成功率提升了50%,处理速度快3~5倍.
```

**正确示例** ✅：
```
使用 git 进行版本控制，先运行 `git init` 命令，然后执行 `git commit`。
成功率提升了 50%，处理速度快 3–5 倍。
```

### 示例 2：对话回复

**用户输入**（使用了英文标点）：
```
你好, 请帮我整理一下代码
```

**Claude 回复**（自动使用中文标点）：
```
你好，我来帮你整理代码。请使用 `git status` 查看当前状态，然后运行 `git add .` 添加文件。
```

### 示例 3：引用区分

```
✅ 他说「我们需要运行 `npm install` 命令」
✅ 这个功能被称为「最强大的工具」
✅ 请阅读《代码大全》一书
✅ 设置 `debug: true` 参数
✅ 支持输入／输出重定向
```

## 自动化规则总结

当 Claude 在中文环境中工作时，应自动：

1. ✅ **使用中文标点符号**，即使用户使用了英文标点
2. ✅ **在中英文之间添加空格**
3. ✅ **在数字与中文之间添加空格**
4. ✅ **英文/数字与中文标点之间不加空格**（如：spec，不是 spec ，）
5. ✅ **中文中使用全角斜杠 `／`**（如：任务／项目，不是任务/项目）
6. ✅ **代码内容使用反引号**，不使用直角引号
7. ✅ **普通引用使用直角引号「」**
8. ✅ **数字范围使用 en dash `–`**
9. ✅ **并列项使用顿号、**
10. ✅ **保持专有名词原有格式**

- [ ] 中英文、数字之间是否有空格？

- [ ] 数字范围是否使用了 `–`？
- [ ] 并列项是否使用了顿号、？
- [ ] 专有名词格式是否正确？
